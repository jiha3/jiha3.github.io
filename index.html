Personal Research

Smart Contract Vulnerability Oracle

    I am implementing an oracle, which classifies smart contract vulnerabilities for any transaction or contract address.
    This oracle is based on previous cases on various blockchains. Our goal is to publish a public smart contract oracle for users.
    Currently I am designing a DB design and gatethering fundamental information about each incident including exploit transactions, attack addresses, and flaws in victim addresses.
    There are several classes for smart contract vulnerablities including storage collision, lack of access control, re-entrancy, etc.
    Since smart contracts are open sources, they easily become targets for attackers. It would be a great benefit for defi users to check the platform's stability prior to using it.

PyPI Integrity Verification

    I am working on a DeFi user study.

    Even though it is of great convenience for them, there is no representative verification or evaluation model.
    If an attacker inserts a malicious code into a PyPI package, there is no way to prevent benign users from checking it.
    They would unknowling install these packages onto their personal computers.
    If this package runs onto a PC, it would act as a malware opening a channel with a C2 server.
    Therefore, I want to develop an automated tool to detect malicious packes in PyPI.
