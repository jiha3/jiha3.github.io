Personal Research

Smart Contract Vulnerability Oracle

    I am implementing a "smart contract vulnerability oracle" for any transaction or contract address.
    Since smart contracts are open sources, they easily become targets for attackers. It would be a great benefit for defi users to check the platform's stability prior to using it.
    This oracle is based on previous hacking incidents on various blockchains. To be skillfull, I have analyzed more than 1,000 incidents regardless of its scale.
    I am gatethering fundamental information from each incident including an exploit transaction, attack address, and a root cause in victim address.
    There would be several vulnerability classes including storage collision, lack of access control, re-entrancy, etc.

PyPI Integrity Verification

    Today, countless PyPI packages are easily usable on the Internet.
    Even though it is of great convenience for users, there is no representative verification or evaluation model.
    Therefore, if an attacker inserts a malicious code into a PyPI package, there is no way to prevent benign users from installing it.
    If this package runs onto a PC, it is likely to open a backdoor with a C2 server.
    My goal is to develop an automated tool to detect a malicious code in a PyPI package when a user try to install it.
